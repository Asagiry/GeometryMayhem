## Принятые решения по выбору ресурсов

### Игровой движок: **Godot Engine 4.5**
- **Почему выбран:**  
  - Бесплатный
  - Прост в освоении
  - Удобен для 2D-проектов (а проект предполагает пиксельную стилизацию и вид сверху).  
  - Активно развивается, имеет современный язык GDScript
  - Поддержка встроенной базы данных (SQLite), что упрощает хранение прогресса.  

---

### Система контроля версий: **Git**
- **Почему выбран:**  
  - Де-факто стандарт в командной разработке.  
  - Позволяет нескольким разработчикам параллельно работать над проектом.  
  - Возможность отката к рабочим версиям и контроль за изменениями кода.  
  - Бесплатный хостинг GitHub

---

### Система управления разработкой: **Jira**
- **Почему выбрана:**  
  - Подходит для управления командой с несколькими ролями.  
  - Удобно вести бэклог, спринты и контроль задач.   
---

### Графический редактор: **Aseprite**
- **Почему выбран:**  
  - Специализирован для пиксельной графики.  
  - Лёгкий и удобный в освоении.  
  - Поддерживает анимацию спрайтов, что критично для динамичной игры.  

---

### База данных: **SQLite (встроенная в Godot)**
- **Почему выбрана:**  
  - Лёгкая, встроенная в движок, не требует отдельной настройки.  
  - Отлично подходит для хранения данных о мета-прогрессии (перманентные усиления).  
  - Подходит под масштаб проекта (нет необходимости в тяжёлой СУБД).  

---

## Общая логика выбора
Все ресурсы выбраны исходя из:
- **Лёгкости в освоении** (команда студенческая, сроки ограничены).  
- **Минимальной стоимости** (все инструменты бесплатные или уже доступные).  
- **Оптимальности под жанр** (2D-экшен с пиксельной графикой).  
- **Удобства командной работы** (Git + Jira для синхронизации процессов).
